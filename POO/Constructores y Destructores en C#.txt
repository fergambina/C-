### Constructores
Un constructor es un método especial en una clase que se ejecuta automáticamente cuando se crea una instancia (objeto) de dicha clase. Su principal propósito es inicializar los atributos o campos de la clase y realizar cualquier configuración necesaria para que el objeto esté listo para su uso.

**Características principales:**
- El nombre del constructor debe coincidir exactamente con el nombre de la clase.
- No tienen tipo de retorno, ni siquiera `void`.
- Pueden recibir parámetros, lo que permite distintas formas de inicialización (sobrecarga de constructores).
- Si no se define ningún constructor, C# proporciona uno predeterminado sin parámetros que no realiza ninguna acción específica.

**Ejemplo de constructor:**
```csharp
public class Persona
{
    public string Nombre { get; set; }
    public int Edad { get; set; }

    // Constructor
    public Persona(string nombre, int edad)
    {
        Nombre = nombre;
        Edad = edad;
    }
}
```

### Destructores
Un destructor es un método especial que se ejecuta automáticamente cuando un objeto está a punto de ser destruido por el recolector de basura (garbage collector) de .NET. Su propósito es liberar recursos que el objeto haya utilizado, como archivos, conexiones a bases de datos, etc.

**Características principales:**
- Solo puede haber un destructor por clase.
- Se define con el símbolo `~` seguido del nombre de la clase.
- No pueden ser llamados explícitamente ni tienen parámetros.
- Su ejecución no es inmediata ni garantizada en el momento exacto en que el objeto deja de ser referenciado.

**Ejemplo de destructor:**
```csharp
public class Recurso
{
    ~Recurso()
    {
        // Código para liberar recursos no administrados
    }
}
```

En la práctica, para liberar recursos de manera más predecible, se recomienda implementar el patrón `IDisposable` en lugar de depender únicamente del destructor.
